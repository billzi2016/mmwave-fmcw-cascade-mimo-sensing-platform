# 项目说明

这一页说明如何维护和扩展项目，同时保持基本结构稳定。

## 保持处理和可视化分离

处理代码应该产出结构化结果。可视化代码应该读取这些结果，并生成图片、视频或归一化资产。

这样做有几个好处：

- 重新渲染结果时不需要重新计算 FFT。
- 修改显示样式时不需要碰原始数据解析。
- 同一份处理结果可以导出多种展示形式。

## 明确数组形状

雷达代码里很多问题来自维度理解错误。新增代码时，应在关键变换附近写清数组形状。

常见例子：

- `(frame, range, doppler, antenna)`
- `(sample, chirp, virtual_rx)`
- `(frame, points, fields)`

## 在布局变化处加注释

注释最有价值的位置通常是：

- reshape，
- transpose，
- 通道映射，
- 文件分组，
- 校准，
- FFT 轴选择，
- 坐标转换。

这些地方只看代码很难马上理解意图。

## 文档逐步扩展

新增输出时，建议按这个顺序更新文档：

1. 先写简短概览。
2. 补充输入和输出文件。
3. 说明处理链路。
4. 写清形状、单位和假设。

目标是持续清晰，而不是每次都大规模重写。
