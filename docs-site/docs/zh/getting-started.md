# 快速开始

这一页用最短路径帮助你理解仓库。

它不是所有脚本的完整命令手册，而是先说明目录之间的关系、应该期待什么输出，以及接下来该打开哪个部分。

## 仓库结构

代码围绕两套雷达系统组织：

```text
mmwave-fmcw-cascade-mimo-sensing-platform/
├── DEMO/
├── TI-IWR2243-Cascade/
│   ├── Processing/
│   └── Visualization/
├── TI-IWR6843/
│   ├── Processing/
│   └── Visualization/
└── docs-site/
```

`Processing` 目录负责把原始雷达数据处理成结构化数组、热力图和点云数据。`Visualization` 目录负责把处理结果导出为图片、视频和归一化资产。

顶层拆分是有意设计的：

- `DEMO/` 展示比较直观的输出效果。
- `TI-IWR2243-Cascade/` 包含四芯片级联路径。
- `TI-IWR6843/` 包含单芯片路径。
- `docs-site/` 包含这套 MkDocs 文档站。

每个雷达目录里又拆成 `Processing` 和 `Visualization`。前者回答“原始采样如何变成结构化数组”，后者回答“结构化数组如何变成可读结果”。

## 推荐阅读顺序

如果你刚开始看这个项目，可以按这个顺序：

1. 先看 `DEMO/`，了解最终结果长什么样。
2. 看仓库根目录 README，理解两条雷达路径。
3. 根据关注的雷达型号进入对应 Processing README。
4. 根据需要查看 Visualization README。
5. 需要慢一点理解处理链路时，再看这套文档站。

如果你已经知道自己使用哪种雷达，可以直接进入对应路径。如果你想比较两条路径，先看 **系统概览** 再读代码会更清楚。

## 最小理解模型

这个项目把原始雷达采样逐步转换成更容易理解的结果：

```text
原始采样
-> 拆帧和通道重建
-> 校准
-> Range FFT
-> Doppler FFT
-> Angle FFT / AoA
-> 热力图和点云
-> 图片、视频、H5、NPZ
```

不需要一开始就理解所有信号处理细节。先弄清输入目录、输出目录和目标结果类型，再逐步深入。

## 选择哪条路径

如果数据来自四芯片级联采集，使用 **TI-IWR2243 Cascade**。你通常会看到多个设备文件，并且可能需要校准 `.mat` 文件。这条路径更复杂，因为代码需要让多个设备的数据保持时间对齐和相位一致。

如果数据来自单芯片采集，使用 **TI-IWR6843**。它通常包含 XML 参数和 BIN 原始采样。这条路径更容易跟读，适合先理解拆帧、FFT 和点云生成。

## 输入输出检查清单

运行处理脚本前，先检查输入：

- 对 IWR2243 Cascade，确认 `master`、`slave1`、`slave2`、`slave3` 的 data 和 idx 文件都存在。
- 对 IWR2243 Cascade，确认是否需要提供校准 `.mat` 文件。
- 对 IWR6843，确认输入目录里同时有 XML 参数文件和 BIN 原始采样文件。
- 对可视化脚本，确认 H5、NPZ 或 MAT 文件已经由处理阶段生成。

运行后再检查输出：

- 如果只导出数据，应看到 H5 或 NPZ 文件。
- 如果启用图片渲染，应看到帧目录或 PNG 文件。
- 如果启用视频渲染，应在逐帧图片生成后看到 MP4 文件。

## 在哪里运行命令

每个处理或可视化子项目都有自己的 `main.py`。

- 处理命令通常在 `Processing/` 目录运行。
- 可视化命令通常在 `Visualization/` 目录运行。
- 如果希望一次运行 IWR2243 级联处理和点云可视化，可以使用 `TI-IWR2243-Cascade/main.py`。

## 第一次排查问题

如果结果看起来不对，先做简单检查：

1. 确认输入路径指向正确采集目录。
2. 确认帧数是否合理。
3. 先确认输出文件已经生成，再判断图像质量。
4. 对热力图，检查距离上限是否把有效 bin 截掉。
5. 对点云，检查空间裁剪范围或固定点数采样是否删掉了太多点。

这些检查通常比直接改信号处理参数更快。
